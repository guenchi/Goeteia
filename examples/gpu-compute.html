100,000 particles, zero CPU physics — on WebGPU

The physics is a @compute shader: one dispatch per frame integrates gravity over a storage buffer of (pos, vel) pairs and respawns the fallen, then the same buffer feeds the render pass as a per-instance vertex stream (gpu-storage! carries both usages). The CPU writes 16 bytes of uniforms per frame and nothing else. This is the transform-feedback trick, the WebGPU way. Needs a WebGPU browser. See gpu-compute.ss.
